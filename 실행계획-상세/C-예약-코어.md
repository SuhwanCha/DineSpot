# C 담당 문서 — 예약 코어 (개발~QA)

## 1. 미션
- 중복 예약 없는 예약 코어 완성.
- 고객/점주 관점 핵심 사용자 여정(예약 생성~변경~취소) 보장.

## 2. 담당 기술 스택
- Go gRPC, MySQL transaction, Redis distributed lock
- 상태 머신 패턴(예약 상태 전이 강제)
- 부하테스트: k6

## 3. 제공/소비 인터페이스
- 제공: Reservation API, Availability API, ReservationEvent
- 소비: A 인증 컨텍스트, B 좌석/영업시간, D 웨이팅 이벤트

## 4. 주차별 상세 작업
- W1: 예약 상태 머신/락 전략 코드화
- W2: 예약 생성/조회/취소 API 1차
- W3: 슬롯 계산/충돌방지 로직 고도화
- W4: 예약 변경/승인 플로우 + FE 연동
- W5: D와 이벤트 연동(노쇼/취소 시 웨이팅 승격)
- W6: 장애복구(재시도/멱등키) 구현
- W7: 동시성 부하 테스트, 병목 제거
- W8: 회귀/결함 수정
- W9: UAT 실데이터 기반 시나리오 점검
- W10: 운영 추적 지표(성공률/지연/충돌차단율) 튜닝

## 5. 중간 교차 점검 포인트
- W2 수요일: C↔B 가용 슬롯 계산 기준 검증
- W3 금요일: C↔A 사용자 권한별 예약 가능 정책 검증
- W5 수요일: C↔D 이벤트 순서/중복 처리 검증

## 6. 완료 기준
- 중복 예약 0건
- 예약 API p99 200ms 이내
- 멱등성 위반 0건
