# D 담당 문서 — 웨이팅/알림 (개발~QA)

## 1. 미션
- 실시간 웨이팅 경험 제공(순번 신뢰성/지연 최소화).
- 이벤트 기반 알림 파이프라인 안정화.

## 2. 담당 기술 스택
- Redis Streams + Consumer Group
- SSE(기본) / WebSocket(확장)
- 알림 어댑터 계층(SMS/Email/Push)

## 3. 제공/소비 인터페이스
- 제공: Waiting API, QueueEvent, Notify API
- 소비: C 예약 이벤트, A 사용자 연락채널 정책, E 알람 연동

## 4. 주차별 상세 작업
- W1: 웨이팅 큐 모델/순번 계산 로직 구현
- W2: 등록/호출/취소 API + 관리자 호출 기능
- W3: 실시간 순번 전파(SSE) 구현
- W4: 알림 템플릿/재시도/중복방지 구현
- W5: C 이벤트 연동(예약 취소 시 자동 승격)
- W6: 네트워크 단절/복구 시나리오 처리
- W7: 지연/정합성 성능 테스트
- W8: 회귀/결함 수정
- W9: UAT 및 알림 품질 튜닝
- W10: 운영 알림 실패 대시보드 고도화

## 5. 중간 교차 점검 포인트
- W3 수요일: D↔FE 실시간 이벤트 페이로드 검증
- W5 금요일: D↔C 이벤트 보상 트랜잭션 검증
- W6 수요일: D↔E 알림 실패 알람 기준 검증

## 6. 완료 기준
- 순번 정합성 오류 0건
- 이벤트 전파 지연 1초 이내
- 알림 중복 발송률 < 0.1%
