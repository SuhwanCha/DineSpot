# B 담당 문서 — 레스토랑/메뉴/운영정책 (개발~QA)

## 1. 미션
- 점주가 매장 운영정보를 안정적으로 관리할 수 있는 도메인 구축.
- C 예약 엔진이 신뢰할 수 있는 매장/좌석/운영시간 데이터 제공.

## 2. 담당 기술 스택
- Go gRPC + MySQL(정규화 스키마)
- 이미지 저장: S3 presigned URL
- FE: 점주 관리 콘솔(Next.js)

## 3. 제공/소비 인터페이스
- 제공: Restaurant/Menu/SeatPolicy API
- 소비: A 권한 컨텍스트, E 감사로그

## 4. 주차별 상세 작업
- W1: 매장/메뉴/좌석 스키마 구현 및 마이그레이션
- W2: 매장 CRUD + 메뉴 CRUD 1차
- W3: 운영시간/휴무일/좌석정책 API
- W4: 점주 관리 UI + 입력검증
- W5: C 예약엔진과 좌석/시간 참조 연동
- W6: 성능 인덱스 튜닝, 검색/정렬 최적화
- W7: 테넌시 격리 테스트(교차 접근 차단)
- W8: 회귀 테스트/결함 수정
- W9: UAT 데이터 정합성 이슈 대응
- W10: 운영자 요청 대응용 관리 쿼리 정리

## 5. 중간 교차 점검 포인트
- W2 월요일: B↔C 좌석 capacity/slot 포맷 고정
- W4 수요일: B↔A 점주 권한 범위 점검
- W6 금요일: B↔E KPI 집계용 필드 누락 점검

## 6. 완료 기준
- 테넌시 위반 0건
- 메뉴/매장 CRUD 성공률 99.9%+
- 예약 참조 데이터 불일치 0건
